3 Pendekatan Token System

Base 13px tetap. Pilih struktur token yang paling pas untuk dashboard ERP.

A

Strict Tailwind — 6 tokens

xs · sm · base · lg · xl · 2xl — persis rasio Tailwind UI

--fs-xs : 0.75rem → 9.75px (badges, timestamps)
--fs-sm : 0.875rem → 11.4px (table headers, nav)
--fs-base: 1rem → 13px (body, table rows)
--fs-lg : 1.125rem → 14.6px (section titles)
--fs-xl : 1.25rem → 16.25px (page titles)
--fs-2xl : 1.5rem → 19.5px (KPI numbers)
✓ Standard, predictable
✗ Tidak ada step di bawah xs — badge label ERP sering butuh 9px ke bawah
B

Extended Tailwind — 7 tokens RECOMMENDED

2xs · xs · sm · base · lg · xl · 2xl — tambah 2xs untuk elemen ERP yang sangat dense

--fs-2xs : 0.6875rem→ 8.9px (badge status, tiny tag)
--fs-xs : 0.75rem → 9.75px (timestamps, breadcrumb)
--fs-sm : 0.875rem → 11.4px (table header, nav item)
--fs-base: 1rem → 13px (body, table row)
--fs-lg : 1.125rem → 14.6px (section title)
--fs-xl : 1.25rem → 16.25px (page title)
--fs-2xl : 1.5rem → 19.5px (KPI number)
✓ Cover semua use-case ERP termasuk badge micro-text
✓ Mapping 1:1 ke semua current font-sizes
✗ Satu token lebih dari standard Tailwind
C

Simplified — 5 tokens

xs · sm · base · lg · xl — hapus extreme sizes, scale paling simple

--fs-xs : 0.6875rem→ 8.9px (badges, tiny)
--fs-sm : 0.8125rem→ 10.6px (secondary)
--fs-base: 1rem → 13px (body)
--fs-lg : 1.125rem → 14.6px (section)
--fs-xl : 1.375rem → 17.9px (page title)
✓ Paling simple, paling mudah diingat
✗ KPI numbers tidak punya token besar — harus hardcode
✗ Gap sm→base terlalu besar, kehilangan nuansa hierarchy
Rekomendasi: B — ERP dashboard butuh micro-text untuk status badge (contoh: 8-9px "Submitted", "Draft"). Tanpa --fs-2xs, badge terpaksa hardcode. Semua current values (0.65–0.85rem) punya home yang clean di scale ini.
